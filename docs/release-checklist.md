# 发布前检查清单

这份清单适合每次准备交付、演示或上线前执行一次。

## 一、环境与配置

- 确认 `APP_ENV` 正确
- 确认 `SECRET_KEY` 已替换为真实随机值
- 确认 `DATABASE_URL` 指向目标环境数据库
- 确认 `CORS_ORIGINS` 已按目标域名配置
- 确认生产环境 `AUTH_COOKIE_SECURE=true`
- 确认 `DEBUG=false`
- 确认 `LOG_LEVEL=INFO` 或更保守
- 确认 LLM 相关配置已填写

## 二、数据库

- 执行 `alembic upgrade head`
- 确认迁移链路没有报错
- 确认目标数据库可连通
- 确认关键表存在：用户、任务、计划、阶段、学习记录、聊天记录
- 上线前先做一次数据库备份

## 三、质量门槛

建议在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 release-check
```

这一步应至少保证：
- `ruff` 通过
- `mypy` 通过
- `pytest` 通过
- `alembic upgrade head` 通过

## 四、核心功能冒烟测试

- 能注册 / 登录 / 登出
- 聊天能发送消息并保存会话
- 聊天能创建任务或学习计划
- 任务页能查看主任务 / 子任务
- 学习计划页能看到阶段、按天安排、按周安排
- “开始学习 / 结束学习”后统计页能看到学习分钟变化
- 提醒设置能正常保存
- 健康检查接口 `GET /api/health` 正常

## 五、部署检查

- Docker 镜像能成功构建
- Compose 配置能通过校验
- 如果使用 PostgreSQL，确认数据库容器或实例已经就绪
- 确认静态资源正常加载
- 确认应用端口和反向代理配置正确

## 六、上线后观察

- 观察应用启动日志是否正常
- 观察健康检查是否持续返回 `200`
- 观察错误日志是否有异常抬升
- 观察提醒任务是否按预期运行
- 抽查一条学习时长记录是否写入统计
